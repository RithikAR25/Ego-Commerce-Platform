# Product Images Schema — EGO Platform

## Tables

### `product_images`

Gallery-level product images — lifestyle shots and detail shots NOT tied to a specific variant.

| Column | Type | Constraints | Description |
|---|---|---|---|
| `id` | BIGINT | PK, AUTO_INCREMENT | Surrogate primary key |
| `product_id` | BIGINT | FK → `products.id`, NOT NULL | Parent product |
| `cloudinary_public_id` | VARCHAR(300) | NOT NULL | Cloudinary stable asset identifier (e.g. `ego/dev/products/1/gallery/abc123`) |
| `url` | VARCHAR(500) | NOT NULL | Full Cloudinary secure_url (original, no transforms) |
| `alt_text` | VARCHAR(255) | NULLABLE | Accessibility alt text for screen readers + SEO |
| `display_order` | INT | NOT NULL, DEFAULT 0 | Sort order. Lower = displayed first. |
| `created_at` | TIMESTAMP | NOT NULL, NOT UPDATABLE | Auto-set by Hibernate `@CreationTimestamp` |

**Indexes:**
- `idx_product_image_product` on `product_id` — fast image list queries

**Cascade:**
- `ON DELETE CASCADE` via Hibernate `CascadeType.ALL` from `Product.images`. Deleting a product auto-deletes all its gallery images.

---

### `variant_images`

Variant-specific images — all colorway photos for a specific `ProductVariant`.

| Column | Type | Constraints | Description |
|---|---|---|---|
| `id` | BIGINT | PK, AUTO_INCREMENT | Surrogate primary key |
| `variant_id` | BIGINT | FK → `product_variants.id`, NOT NULL | Parent variant |
| `cloudinary_public_id` | VARCHAR(300) | NOT NULL | Cloudinary stable asset identifier (e.g. `ego/dev/products/1/variants/7/black_hero`) |
| `url` | VARCHAR(500) | NOT NULL | Full Cloudinary secure_url (original, no transforms) |
| `alt_text` | VARCHAR(255) | NULLABLE | Accessibility alt text |
| `is_primary` | BOOLEAN | NOT NULL, DEFAULT false | Marks the hero image for listing cards and PDP |
| `display_order` | INT | NOT NULL, DEFAULT 0 | Thumbnail strip sort order |
| `created_at` | TIMESTAMP | NOT NULL, NOT UPDATABLE | Auto-set by Hibernate |

**Indexes:**
- `idx_variant_image_variant` on `variant_id` — fast image list queries

**Cascade:**
- `ON DELETE CASCADE` via Hibernate `CascadeType.ALL` from `ProductVariant.images`. Deleting a variant auto-deletes all its images.

## Entity Relationship

```
products (1)
    │
    └── product_images (N)
            cloudinary_public_id, url, alt_text, display_order

product_variants (1)
    │
    └── variant_images (N)
            cloudinary_public_id, url, alt_text, is_primary, display_order
```

## Primary Image Invariant

Each `ProductVariant` must maintain exactly **one** `VariantImage` with `is_primary = true`.

**Enforcement rules:**
1. First image upload → auto-set `is_primary = true`.
2. Explicit primary set → atomically clears all others via `UPDATE variant_images SET is_primary = false WHERE variant_id = ?`, then sets the new one.
3. Primary image delete → promotes next image (lowest `display_order`) to `is_primary = true` automatically.

There is no database-level `UNIQUE` constraint on `is_primary = true` — enforcement is entirely at the service layer in `VariantImageService`. This is intentional: a UNIQUE partial index on `is_primary = 1` is MySQL-dialect-specific and adds migration complexity for minimal benefit.

---

## Key Design Decisions

### Only `public_id` Stored

Only `cloudinary_public_id` is persisted. Transformation URLs (thumbnail, card, detail, zoom) are generated at response-time by `CloudinaryService.buildTransformationUrls()`.

**Rationale:** If transformation presets change, only the service layer changes — zero data migrations needed.

### No Eager Collections on Product/Variant

`ProductImage` and `VariantImage` collections use `FetchType.LAZY`. They are loaded only when explicitly accessed in a Hibernate session — avoiding N+1 issues in paginated product listing queries.

### `display_order` Not Unique

`display_order` is not unique-constrained. If two images have the same order value, their relative order is stable (database insertion order as tiebreaker). The UI should always send a complete, unique ordering via the reorder API.

## Hibernate DDL (Reference)

```sql
-- Auto-generated by Hibernate ddl-auto=update
CREATE TABLE product_images (
    id                    BIGINT AUTO_INCREMENT PRIMARY KEY,
    product_id            BIGINT NOT NULL,
    cloudinary_public_id  VARCHAR(300) NOT NULL,
    url                   VARCHAR(500) NOT NULL,
    alt_text              VARCHAR(255),
    display_order         INT NOT NULL DEFAULT 0,
    created_at            TIMESTAMP NOT NULL,
    CONSTRAINT fk_pi_product FOREIGN KEY (product_id)
        REFERENCES products(id) ON DELETE CASCADE,
    INDEX idx_product_image_product (product_id)
);

CREATE TABLE variant_images (
    id                    BIGINT AUTO_INCREMENT PRIMARY KEY,
    variant_id            BIGINT NOT NULL,
    cloudinary_public_id  VARCHAR(300) NOT NULL,
    url                   VARCHAR(500) NOT NULL,
    alt_text              VARCHAR(255),
    is_primary            BOOLEAN NOT NULL DEFAULT false,
    display_order         INT NOT NULL DEFAULT 0,
    created_at            TIMESTAMP NOT NULL,
    CONSTRAINT fk_vi_variant FOREIGN KEY (variant_id)
        REFERENCES product_variants(id) ON DELETE CASCADE,
    INDEX idx_variant_image_variant (variant_id)
);
```
